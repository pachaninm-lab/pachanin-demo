# Лендинг: доставка заявок на Gmail

Документ описывает, как заявки с лендинга `pachanin-demo-landing` отправляются отдельным письмом на Gmail.

## Что работает

Endpoint:

```txt
POST /api/pilot-request
```

Принимает заявки из формы внизу лендинга.

Каждая заявка формируется как отдельное письмо с темой:

```txt
Прозрачная Цена — заявка — <роль или компания>
```

## Канал доставки

Используется Resend.

Переменные окружения в Vercel:

```txt
RESEND_API_KEY=
LEAD_TO_EMAIL=pachaninm@gmail.com
RESEND_FROM_EMAIL=
```

`RESEND_FROM_EMAIL` должен быть подтверждённым отправителем в Resend.

## Что отправляется в письме

```txt
Время
Источник
Цель
Имя
Компания
Роль
Телефон
Email
Регион
Культура и объём
Ключевой риск
Сценарий
Срок
Описание сделки
```

## Что убрано

- Мини-чат с лендинга.
- Telegram-доставка.
- Webhook-доставка.
- Сценарий копирования сводки пользователем.

## Важно

Не писать на лендинге, что интеграции с банками, ФГИС «Зерно», СДИЗ, ЭДО или CRM завершены, пока нет боевого доступа, договора и подтверждённого сценария.

Текущий статус лендинга: сбор заявок для проверки интереса и controlled pilot.
