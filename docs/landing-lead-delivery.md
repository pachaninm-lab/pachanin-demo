# Лендинг: доставка лидов

Документ описывает, как заявки и ответы мини-чата с лендинга `pachanin-demo-landing` доставляются владельцу проекта.

## Что уже работает

Endpoint:

```txt
POST /api/pilot-request
```

Принимает лиды из:

- мини-чата проверки интереса;
- быстрой заявки внизу лендинга.

Если внешние каналы не подключены, endpoint всё равно принимает заявку и пишет структурированный лид в Vercel server logs.

Клиент получает статус `accepted: true` и текст лида `leadText`, чтобы посетитель мог скопировать сводку или отправить её на почту.

## Каналы доставки

### 1. Telegram

Переменные окружения в Vercel:

```txt
TELEGRAM_BOT_TOKEN=
TELEGRAM_CHAT_ID=
```

После настройки каждый лид уходит в Telegram сообщением.

### 2. Email через Resend

Переменные окружения в Vercel:

```txt
RESEND_API_KEY=
LEAD_TO_EMAIL=pachaninm@gmail.com
RESEND_FROM_EMAIL=
```

`RESEND_FROM_EMAIL` должен быть подтверждённым отправителем в Resend.

### 3. Webhook для Google Sheets / CRM / Make / Zapier

Переменная окружения в Vercel:

```txt
PILOT_WEBHOOK_URL=
```

Payload:

```json
{
  "lead": {
    "createdAt": "ISO date",
    "source": "landing_interest_chat",
    "intent": "Проверить интерес к использованию платформы",
    "interestLevel": "горячий интерес",
    "interestSummary": "...",
    "name": "...",
    "company": "...",
    "phone": "...",
    "email": "...",
    "role": "...",
    "region": "...",
    "cropVolume": "...",
    "risk": "...",
```

## Важно

Не писать на лендинге, что интеграции с банками, ФГИС «Зерно», СДИЗ, ЭДО или CRM завершены, пока нет боевого доступа, договора и подтверждённого сценария.

Текущий статус лендинга: сбор интереса и заявок для controlled pilot.
