# ADR-001: Хранение денег в целых числах (копейки)

**Статус:** Принято  
**Дата:** 2026-01-15  
**Авторы:** Backend Team  

## Контекст

Платформа обрабатывает финансовые транзакции — escrow, комиссии, settlements. Неправильное хранение денег может привести к потере средств из-за floating-point ошибок.

## Решение

Все суммы хранятся как `BIGINT` в **копейках** (1 рубль = 100 копеек).

```typescript
// Правильно
totalKopecks: 1_000_000  // 10 000 руб

// Неправильно (никогда)
totalRub: 10000.00  // Float precision loss риск
```

**Поле `totalRub`** (Float) существует только для обратной совместимости и display-purposes. Все расчёты производятся через `totalKopecks`.

## Последствия

- `+` Точность: целочисленная арифметика без потерь
- `+` Совместимость с банковскими API (работают в копейках)
- `+` Инвариант `SUM(debit) = SUM(credit)` проверяем точно
- `-` Нужна конвертация при отображении пользователю (`kopecks / 100`)
- `-` Максимум: 9.2×10^18 копеек (~92 квадриллиона рублей) — достаточно

## Альтернативы, отклонённые

- **Decimal/BigDecimal** — не поддерживается нативно в SQLite/PostgreSQL как integer, требует cast
- **Float** — неприемлемо: `0.1 + 0.2 !== 0.3`
- **String** — избыточно для математики
