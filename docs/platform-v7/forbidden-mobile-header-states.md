# Platform v7 — единая шапка и запрещённые состояния

Единственная допустимая шапка для всей `platform-v7`: фиксированная `PlatformV7UnifiedHeader`.

## Норма для всех страниц без исключения

- слева канонический `BrandMark`;
- рядом текст `Прозрачная Цена`;
- справа только три действия: `язык`, `Войти/Выход`, `бургер`;
- на публичных страницах основная кнопка — `Войти`;
- в рабочих кабинетах основная кнопка — `Выход`;
- все остальные действия находятся внутри role-aware burger drawer;
- burger drawer разделён на группы, а не является свалкой ссылок;
- шапка всегда `fixed`;
- контент начинается ниже spacer и не заходит под шапку;
- внешний вид одинаковый при cold load, client navigation, back/forward и refresh;
- нижняя role-панель может оставаться как быстрый role-specific dock, но не заменяет burger drawer.

## Запрещено

### Публичные страницы

- шапка с иконкой колоса вместо фирменного `BrandMark`;
- шапка без логотипа;
- шапка только с текстом `Прозрачная Цена` и кнопкой `Войти`;
- центрированный текст бренда без логотипа слева;
- кнопка `Выход`, если пользователь находится на публичной странице и ещё не вошёл;
- разный вид шапки до и после refresh/hydration;
- разные размеры логотипа на `/platform-v7`, `/login`, `/register`, `/contact`, `/demo`, `/docs`.

### Рабочие кабинеты

- больше одного hamburger/menu в шапке;
- правая группа из набора `поиск + документы + уведомления + помощь + калькулятор + выход + тема`;
- красный выход как единственное большое действие в mobile-шапке без бренда и burger;
- переключатель темы в основной строке mobile header;
- уведомления/помощь/блокнот/калькулятор, если они сжимают или раздвигают шапку;
- отсутствие нижней role-панели при первом переходе из login;
- появление шапки/низа только после второго refresh.

## QA-критерий

Для каждой страницы: открыть cold load → войти/перейти → сделать refresh. До и после refresh должны совпадать:

- логотип;
- размеры иконок;
- расположение language / main action / burger;
- смысл main action: `Войти` на public, `Выход` в кабинете;
- набор правых действий;
- нижняя навигация;
- отсутствие горизонтального скролла;
- контент не заходит под шапку.
