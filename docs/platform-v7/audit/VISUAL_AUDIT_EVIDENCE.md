# Visual Audit (evidence-based) — platform-v7 / «Прозрачная Цена»

Дата: 2026-06-15. Метод: измерения по всем компонентам/стилям platform-v7
(156 + 61 компонентов, 181 маршрут, 17 CSS-файлов / 4013 LOC). Дополняет
`VISUAL_UX_AUDIT.md` твёрдыми цифрами. Планка: дорогая B2B/fintech SaaS.

## 1. Тема (dark/light) — токены vs hex

| Метрика | Значение |
|---------|----------|
| Использований токенов `var(--pc-/--p7-)` | 4 886 |
| Всего `#rrggbb` в компонентах | 5 135 |
| из них **token-fallback** `var(--token, #hex)` (ОК-практика) | 3 235 (≈63%) |
| `rgba(...)` в компонентах | 1 503 |
| Файлов с hex | 273 |

Интерпретация: доминирующий паттерн — **токен с hex-fallback** (3 235 из 5 135),
это корректно для SSR/тёмной темы. Остаток (~1 900 standalone hex + часть rgba) —
преимущественно статический CSS в `<style>` и точечные акценты.

**VIS-001 (MEDIUM):** токенизация не завершена на всём объёме — 273 файла
содержат hex. Денежные/банковские экраны уже закрыты (UX-001/UX-002). Fix:
продолжить миграцию theme-sensitive поверхностей в токены; Test: расширить
`platformV7MoneyBankThemeTokens`-guard на новые группы. Status: частично закрыто,
остальное — follow-up (in-scope, итеративно).

## 4. Слои / типографика

- **VIS-002 (LOW):** `z-index` — 22 уникальных значения, максимум **9999**;
  ввести единую шкалу токенов z-index во избежание «гонки наслоений».
  `position: fixed/sticky` — 43 (overlay/headers), приемлемо.
- **VIS-003 (INFO):** 663 объявления `fontSize: 10/11px` — это **микро-лейблы**
  (uppercase eyebrows/captions), что соответствует премиум-системе; важные данные
  (суммы/KPI/hero) — 18–32px (≥16, как требует спецификация). Нарушения «важные
  данные <16px» не обнаружено.

## 7. Ограничение (честно)
Программный axe-скан и пиксельный/контрастный QA требуют живого рендера:
`axe-core` напрямую не установлен, `@axe-core/playwright` требует браузер-сборку,
недоступную в песочнице; Vercel заблокирован (HTTP 402), Netlify активен —
route-smoke зелёный, но визуальная глубина (контраст по WCAG в реальном лэйауте,
скриншоты 390×844 dark/light) — owner-side на живом стенде. Структурный визуальный
аудит (токены, overflow, адаптив, a11y-атрибуты, типографика, слои) — выполнен по
коду и приведён выше.

## Вердикт (визуальный)
Сильная премиум-система, надёжные overflow/адаптив-гарды, хорошая a11y-база,
читаемые денежные/банковские экраны. Визуальный долг управляем и точечен
(VIS-001 токенизация — главный, итеративный; VIS-002/004 — мелочи). Пиксельный/
контрастный live-QA — owner-side (живой браузер/стенд).
