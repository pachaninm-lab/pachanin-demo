# UX-003 — CTA hierarchy convention (platform-v7)

Дата: 2026-06-15. Назначение: зафиксировать конвенцию иерархии действий, чтобы
закрыть находку UX-003 («на части кокпитов 2–3 CTA, размыт главный путь») без
рискованной массовой правки страниц.

## Правило

На каждом кокпите/экране — **ровно один primary CTA** (главное следующее
действие), остальные действия визуально подчинены:

| Уровень | Вариант компонента | Когда |
|---------|--------------------|-------|
| Primary | `PremiumCtaButton` (default / зелёный градиент) | одно главное действие экрана |
| Secondary | `PremiumCtaButton variant="ghost"` | альтернативный путь (например «открыть сделку») |
| Tertiary | ссылка/`variant="ghost"` в сайдбаре/футере | навигация, не действие |

Принципы:
- Primary отвечает на вопрос «что сделать прямо сейчас, чтобы продвинуть сделку».
- Не более одного primary в зоне hero.
- Деструктивные/останавливающие действия — `variant="danger"`, не primary.
- На мобиле primary располагается первым и имеет touch-target ≥44–56px.

## Статус по ролям (текущее)

Кабинеты используют `PremiumCtaButton` (primary) + `ghost` для вторичных —
структура уже верная; находка UX-003 о том, чтобы держать **один** primary в
hero. Это конвенция-гайд для последующих точечных правок страниц, без изменения
маршрутов/ролей/тестов.

## Регрессионная защита

Постоянный guard `tests/unit/platformV7NoFakeLiveCopy.test.ts` фиксирует запрет
fake-live копий по всему platform-v7. Для CTA-иерархии рекомендуется при правке
конкретной страницы добавлять локальный тест «один primary CTA в hero».
