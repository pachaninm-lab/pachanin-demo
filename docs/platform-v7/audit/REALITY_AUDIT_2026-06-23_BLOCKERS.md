# platform-v7 blocker model — 2026-06-23

## Blocker fields

Each cabinet blocker should include:

- blocker label;
- affected flow;
- money exposure;
- responsible party;
- missing evidence or confirmation;
- next safe action;
- maturity boundary.

## Maturity boundary values

- controlled-pilot;
- pre-integration;
- external confirmation required;
- operator review required;
- audit draft only.

## Purpose

The blocker model keeps UI useful for execution without presenting incomplete runtime domains as live.
