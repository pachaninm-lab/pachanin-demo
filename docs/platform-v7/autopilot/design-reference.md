# Living Deal Interface — дизайн-референс (от владельца, 2026-06-10)

Источник: 11 макетов владельца (product entry, seller, buyer, bank, operator,
lab, elevator, executive, driver mobile). Применять при VP-5…VP-8 / UX / HX
слайсах. Это направление, не пиксель-перфект.

## Система

- Светлая тема по умолчанию: молочно-кремовый фон страниц, белые мягкие карточки
  с тонкой границей и лёгкой тенью, скруглением ~14–18px.
- Примари-цвет — спокойный зелёный (как текущий `--pc-accent` #0A7A5F),
  один зелёный главный CTA на экран; secondary — белые кнопки с границей.
- Левый сайдбар роли: лого «Прозрачная Цена», разделы роли, низ — профиль.
  Верхняя полоса: переключатель контура, глобальный поиск, уведомления, аватар.
- Типографика строгая, крупные суммы tabular-nums; статусы — цвет + текст.

## Ключевые паттерны по макетам

- Product entry: hero на фоне пшеничного поля, «Одна сделка. Полный контроль.»,
  цепочка исполнения иконками (Условия→Документы→Рейс→Приёмка→Качество→Деньги→
  Спор/Закрытие), CTA «Выставить партию» / «Создать запрос на закупку» /
  «Открытый просмотр», грид ролей с короткими описаниями. Без фейковых сумм.
- Role cockpit (seller/buyer): сверху денежная сводка (сумма + Money Gate Ring
  с долями), активная партия, Document Stack со статусами и влиянием на деньги,
  колонка справа: Smart Collapse «Почему это важно», следующее действие,
  Action Receipt.
- Bank Clean Room: стерильно — Money Gate Ring, основание выплаты, строгие
  таблицы документов/журнала, ручная проверка; никакого glow.
- Operator: очередь активных сделок с SLA-колонкой, кластеры проблем,
  правая колонка — быстрые действия и Action Receipt.
- Lab: очередь образцов, протокол с показателями и quality delta барами
  (норма/факт/допуск), итог по образцу, Smart Collapse-подсказка.
- Elevator (приёмка): очередь прибытий, карточка машины с фото и QR,
  весовые значения (заявлено/брутто/нетто) крупно, расхождение красным,
  действия «Принять решение» / «Зафиксировать расхождение».
- Executive Signal Wall: 4–5 крупных сигналов (деньги в блоке, споры,
  главный блокер, время до оплаты), ниже — только нужные графики.
- Driver Mission Mode (mobile): офлайн-баннер сверху, карточка текущего рейса
  с route pulse, большая зелёная «Начать рейс», ряд крупных кнопок
  (Я прибыл / Фото / Документы / Проблема), фото-чеклист прогрессом,
  Smart Collapse и Action Receipt снизу.

## Ограничения (из ФИНАЛЬНОГО ТЗ)

HX-слой — только после runtime binding базовых экранов; анимации функциональные,
reduced-motion обязателен; статус всегда дублируется текстом; один главный CTA.
