# Mobile shell regression pass — platform-v7

Дата: 2026-06-17
Контур: platform-v7 / Netlify-only

## Проверенный дефект

Нижняя панель была безопасной по RBAC, но некачественной по UX:

- часть кнопок была фейковой и вела на тот же экран;
- часть кнопок дублировала шапку;
- кнопка «Сигналы» дублировала уведомления;
- кнопка «Меню» присутствовала даже там, где дополнительного меню не требовалось;
- старый глобальный колокольчик показывал общие уведомления, не уведомления роли;
- логотип был перехвачен как переход в домашний экран роли, хотя пользовательский сценарий требует публичную главную с описанием платформы и входом.

## Исправления

### Нижняя панель

Принцип: только реальные маршруты и действия. Если отдельного маршрута нет, кнопка не показывается.

- Убраны фейковые кнопки «Фото», «Вес», «Акты», «Качество», «Протокол», которые вели на тот же экран.
- Убрана кнопка «Сигналы» из нижней панели.
- Кнопка «Меню» показывается только если у роли есть дополнительные пункты в safe drawer, которых нет в нижней панели.
- ИИ-помощник сохранён как отдельная кнопка для каждой роли.

### Уведомления

- Старый глобальный колокольчик AppShell скрыт.
- Добавлен новый ролевой колокольчик.
- Уведомления берутся из `NOTICES_BY_ROLE` и ведут только в разрешённые маршруты текущей роли.

### Логотип

- Логотип в шапке и drawer всегда ведёт на `/platform-v7`.
- Он больше не ведёт в `/platform-v7/login` и не ведёт в домашний экран роли.

### ИИ-помощник

- ИИ берёт активную роль из `sessionStorage` ключа `pc-v7-active-role`.
- ИИ игнорирует URL-параметр `role`.
- ИИ фильтрует все suggested actions через role allowlist.
- ИИ не может предложить действие чужой роли.

## Причина подвисаний

Основные источники тяжести и визуального подвисания на мобильном:

1. Двойной shell: старый `AppShellV4` остаётся в DOM, а новый `PlatformV7ShellUxController` перехватывает его через portal/CSS.
2. Старые глобальные элементы AppShell: drawer, notification panel, legacy bottomnav, role switcher — часть скрывается CSS, но всё равно рендерится.
3. `PublicEntryCleanup` использует `MutationObserver` по `document.body`, что на сложных страницах может давать лишние sync-проходы.
4. Большие role/runtime страницы рендерят много карточек, inline styles и локальных массивов без виртуализации.
5. Страница `/platform-v7/ai` раньше строила широкий action-config с чужими ссылками; теперь сужена, но всё ещё client-side.

## Что закрыто сейчас

- Ломание нижней панели из-за фейковых кнопок.
- Дублирование уведомлений нижней панелью.
- Глобальные уведомления вместо ролевых.
- Логотип в неправильный экран.
- ИИ-действия чужой роли.
- Часть визуальной нагрузки от лишних bottom actions.

## Что остаётся как технический долг

Для дальнейшего ускорения без костылей нужно:

1. Удалить старый bottomnav/drawer/notification logic из `AppShellV4`, а не скрывать CSS.
2. Перенести safe drawer, dock и notices в базовый shell вместо overlay-controller.
3. Ограничить `MutationObserver` в `PublicEntryCleanup` только публичной главной.
4. Убрать inline-heavy визуал из самых тяжёлых экранов в CSS/classes.
5. Сделать browser-level smoke на 390×844 для всех ролей.

## Manual QA чек после Netlify deploy

Для каждой роли:

- Логотип → `/platform-v7`.
- Выход → `/platform-v7` и очищает роль.
- Уведомления → только уведомления роли.
- Нижняя панель → только реальные маршруты и ИИ.
- Меню → только если есть дополнительные функции.
- ИИ → только действия текущей роли.
- Прямые чужие URL → редирект на текущий кабинет или единый вход.
