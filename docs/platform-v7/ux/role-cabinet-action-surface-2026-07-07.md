# Role cabinet action surface

Цель: заменить ощущение перегруженного кабинета на рабочий экран выбора действия.

Правило: пользователь сначала выбирает намерение, затем попадает в нужный экран сделки, документов, денег, рейса или спора.

Что реализовано в компоненте RoleExecutionSummary:

- блок выбора действия по роли;
- блок важных условий;
- блок продолжения начатой работы;
- сохранение существующих ссылок на рабочие контуры;
- отсутствие учебного сценария на рабочих экранах.

Проверочный принцип: каждый видимый переход должен отвечать на вопрос, что пользователь сделает после нажатия.
