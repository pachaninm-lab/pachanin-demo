# 152-ФЗ / Data Protection — platform-v7

Дата: 2026-06-15. Зрелость: controlled-pilot. Связано с
`../audit/SOC2_READINESS.md` и Legal DD.

## Что есть (EXISTS)

- **Классификация и доступ к чувствительным полям:** `deal-access-gate.ts`
  (`SensitiveField`: phone, email, exact_address, full_legal_name, bank_details,
  full_documents, responsible_person, driver/carrier_contact, closed_offer_terms),
  `canAccessSensitiveField`/`maskSensitiveValue`.
- **Контакт-волт:** `contact-vault.ts` — поля шифруются; reveal-политика
  привязана к стадии сделки + риск обхода + одобрение оператора; раскрытие
  логируется в аудит.
- **Минимизация по ролям:** элеватор/лаб/водитель не видят коммерческие/
  финансовые/контактные данные вне своей роли.
- **Доступ-лог:** раскрытие контакта пишет audit-event с причиной.

## Чеклист 152-ФЗ (owner-side)
- [ ] Согласия на обработку ПД (consent) — модель + UI + хранение.
- [ ] Уведомление РКН об обработке ПД.
- [ ] Локализация хранения ПД граждан РФ (ст. 18 152-ФЗ).
- [ ] Право на доступ/исправление/удаление по субъекту.
- [ ] Политика хранения/удаления ПД (TTL), отдельно от evidence-retention.
- [ ] DPA с подрядчиками (хостинг/аналитика — Yandex Metrica/Sentry).

## Вердикт
Технический контроль доступа к ПД и минимизация — есть и сильные. Юридические
механизмы (consent, локализация, права субъекта, durable-лог) — owner-side +
точечные future-PR; до них обработка реальных ПД не должна включаться.
