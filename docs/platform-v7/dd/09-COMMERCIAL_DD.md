# Commercial Due Diligence — platform-v7 / «Прозрачная Цена»

Дата: 2026-06-15. Зрелость: controlled-pilot. Коммерческие данные (выручка,
воронка, юнит-экономика, контракты клиентов) — owner-side; ниже — что видно из
кода и что нужно подтвердить документами.

## 1. Сегмент и ценностное предложение

- **Рынок:** B2B оптовая торговля зерном (РФ), agri-fintech. Стороны: продавцы
  (КФХ/элеваторы), покупатели, перевозчики/водители, лаборатории, банк,
  оператор.
- **Value-prop (из продукта):** прозрачная цена + контролируемое исполнение
  сделки (деньги привязаны к документам/качеству/рейсу/спору) + ролевая
  изоляция + основание банковской выплаты без «ручного выпуска денег».

## 2. Монетизация (сигналы в коде)

- Есть `deal-financial-terms.ts` (финансовые условия сделки) и доменные удержания/
  резервы/основания. **COMM-001 (owner-side):** конкретная модель комиссии/
  тарификации/выручки в коде как биллинг-движок не реализована — подтверждается
  бизнес-документами (прайс, договоры, комиссия с оборота/подписка).
- Нет платёжного эквайринга платформы (деньги движутся через банк по основанию) —
  снижает регуляторную нагрузку, но требует банковского партнёра.

## 3. Готовность к выходу

- Технически: controlled-pilot-ready (см. Technical/Product DD).
- Коммерчески: **COMM-002 (owner-side):** нет подтверждённых платящих клиентов/
  пилотных контрактов в коде; первая controlled-pilot сделка ещё не проведена.
- Зависимость от партнёров (банк/ФГИС/ЭДО) — критична для запуска выручки.

## 4. Что нужно для коммерческого DD (owner-side)
- [ ] Бизнес-модель и юнит-экономика (комиссия/подписка/take-rate).
- [ ] Пайплайн клиентов и LOI/пилотные соглашения.
- [ ] Договорённости с банком (номинальный счёт/эскроу-механика).
- [ ] Метрики traction (GMV, число сделок) — после пилота.
- [ ] Конкурентный анализ и ценообразование.

## Вердикт (Commercial)
Продуктовая основа под коммерциализацию готова (controlled-pilot), но
коммерческие доказательства (клиенты, выручка, юнит-экономика, банк-партнёр) —
owner-side и не выводятся из кода. Это не дефект кода, а стадия (pre-revenue
pilot).
