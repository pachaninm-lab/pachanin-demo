# Промышленные ярлыки ИИ-помощников

## 1. Назначение

Платформа предоставляет два технически и юридически разделённых контура:

1. **Помощник по платформе** — публичная база знаний на `/platform-v7`.
2. **Помощник сделки** — приватный ролевой помощник в авторизованном `platform-v7`.

Общее визуальное обозначение не означает общую сессию, память, API или набор данных.

## 2. Граница данных

| Контур | Данные | Авторизация | Действия |
|---|---|---|---|
| Публичный | Версионированные публичные знания | Не требуется | Запрещены |
| Приватный | Только серверно разрешённая проекция текущего ЛК и Сделок | Обязательны membership, tenant, role и Deal access | `READ_ONLY` до отдельной command-acceptance |

Публичный помощник не получает:

- идентификатор пользователя;
- membership, tenant, организацию или роль;
- список Сделок;
- документы, деньги, сообщения или audit trail;
- историю приватного помощника;
- cookies и токены авторизованного API как входные данные ответа.

Приватный помощник не использует публичный endpoint как fallback и не заменяет недоступные authoritative-данные публичными знаниями.

## 3. Пользовательские поверхности

### 3.1. Главная

Публичный ярлык находится в левой нижней safe-area. Человеческая поддержка остаётся отдельным каналом справа. Они не заменяют друг друга:

- ИИ объясняет устройство платформы;
- поддержка принимает обращения и контактные запросы.

### 3.2. Авторизованный workspace

Составной ярлык состоит из:

- текстового входа **«Помощник сделки»** в полноэкранный workspace;
- компактного AI-trigger для быстрого открытия контекстной панели.

Оба входа ведут в один приватный ролевой контур, а не создают двух помощников.

## 4. Публичная база знаний

Текущая версия:

```text
public-platform-knowledge-2026-07-17.v1
```

Обязательные домены знаний:

- назначение платформы;
- канонический путь Сделки;
- 12 ролей;
- аукцион;
- логистика, приёмка и лаборатория;
- документы и доказательства;
- деньги и банковский контур;
- спор;
- внешние интеграции;
- безопасность;
- различие публичного и приватного помощников.

Каждая тема должна содержать:

- краткое объяснение;
- проверяемые ключевые факты;
- честную границу зрелости;
- ссылки на публичные разделы;
- безопасные продолжения диалога.

## 5. Управление знаниями

Изменение базы знаний требует:

1. новой версии `PUBLIC_ASSISTANT_KNOWLEDGE_VERSION`;
2. проверки RU / EN / ZH;
3. проверки ссылок на канонические публичные маршруты;
4. сверки зрелости интеграций;
5. unit- и browser-acceptance;
6. exact-head deploy preview;
7. merge только после зелёного scope guard.

Запрещено:

- заявлять адаптер как действующую интеграцию;
- переносить маркетинговую гипотезу в фактический ответ;
- описывать архитектурную готовность как подтверждённую эксплуатацию;
- добавлять персональные, договорные или транзакционные сведения;
- использовать непроверенный перевод только на одном языке.

## 6. API-контракт публичного помощника

```json
{
  "dataMode": "public_knowledge",
  "mode": "read_only",
  "knowledgeVersion": "public-platform-knowledge-2026-07-17.v1",
  "actionAllowed": false,
  "topic": "money",
  "title": "Банковский контур и деньги",
  "answer": "...",
  "facts": [],
  "maturity": "...",
  "confidence": "high",
  "sources": [],
  "suggestions": []
}
```

Инварианты:

- `actionAllowed` всегда `false`;
- endpoint не вызывает внешнюю LLM;
- endpoint не принимает account-, tenant-, role- или Deal-контекст;
- тело и сообщение имеют жёсткие лимиты;
- ответ не кешируется shared-cache;
- cross-site browser POST отклоняется;
- полный текст вопроса не записывается кодом помощника;
- язык источников соответствует языку ответа.

## 7. Threat model

| Угроза | Контроль |
|---|---|
| Попытка получить данные ЛК | У публичного endpoint отсутствуют account tools и private API |
| Prompt injection | Вопрос используется только для выбора фиксированной темы, а не как инструкция к tool execution |
| Подмена зрелости | Каждая тема содержит обязательное поле `maturity` |
| Незаметное смешение режимов | `dataMode=public_knowledge`; приватный режим использует другие API и контракты |
| XSS | React escaping, фиксированные внутренние ссылки, JSON response |
| Cross-site browser abuse | Проверка `Sec-Fetch-Site` для POST |
| Большие запросы | Лимиты тела и сообщения |
| Стоимость внешней модели | Внешний provider отсутствует |
| Массовый трафик | Горизонтально масштабируемый stateless handler; перед высоким публичным трафиком обязателен CDN/WAF rate limit |

Последний пункт является инфраструктурной границей controlled production. Приложение не должно имитировать distributed rate limit через process-memory.

## 8. Наблюдаемость

Клиент отправляет только технические события без текста вопроса:

- `public_platform_assistant_opened`;
- `public_platform_assistant_question` — длина и locale;
- `public_platform_assistant_answer` — topic и confidence;
- `public_platform_assistant_source_opened`;
- `public_platform_assistant_reset`;
- `private_assistant_shortcut_opened`.

До реальных пользователей требуется определить серверные метрики:

- RPS и concurrent requests;
- p50 / p95 / p99 latency;
- 4xx / 5xx;
- доля неизвестных вопросов;
- темы ответов;
- размер запросов;
- CDN/WAF blocks;
- функция и стоимость на 1 000 ответов.

Текст вопроса не должен попадать в analytics по умолчанию.

## 9. SLO

Для controlled production публичного детерминированного помощника:

| Показатель | Цель |
|---|---:|
| Availability | ≥ 99,9% в месяц |
| p95 ответа приложения | ≤ 500 мс без учёта сети пользователя |
| 5xx | < 0,1% |
| Cross-account leakage | 0 |
| External provider calls | 0 |
| Ответы без `knowledgeVersion` | 0 |
| Смешение public/private mode | 0 |
| Непереведённые элементы RU/EN/ZH | 0 |

## 10. Деградация

Если публичный endpoint недоступен:

- главная продолжает работать;
- вход, регистрация и человеческая поддержка продолжают работать;
- ярлык показывает нейтральную ошибку;
- ответ не заменяется выдуманными данными;
- приватный помощник не затрагивается.

Если приватный backend недоступен:

- публичный помощник не используется как fallback;
- приватный помощник завершается fail-closed;
- состояние Сделки не изменяется.

## 11. Приёмка

Обязательные проверки:

- ярлык присутствует только на главной для public mode;
- человеческая поддержка не удалена;
- публичный endpoint не содержит external model calls;
- публичный компонент не вызывает private assistant API;
- приватный ярлык присутствует только в private workspace;
- `/platform-v7/assistant` не получает второй плавающий ярлык;
- RU / EN / ZH;
- 320 / 375 / 390 / 430 CSS px;
- keyboard, Escape, focus trap;
- reduced motion и forced colors;
- exact-file scope guard;
- typecheck, tests, build, security и browser matrix;
- exact-head Netlify deploy preview.

## 12. Честная зрелость

После merge допустимая формулировка:

> «Разделённые публичный knowledge-assistant и приватный ролевой помощник Сделки с версионированными знаниями, mobile-first ярлыками, fail-closed границами и provider-neutral архитектурой».

Недопустимые формулировки:

- «общается с реальными Сделками на главной»;
- «подключён к банку, ФГИС или ЭДО»;
- «автономно выполняет операции»;
- «production-проверен на реальных пользователях»;
- «имеет промышленный distributed rate limit», пока CDN/WAF-контур отдельно не принят.
