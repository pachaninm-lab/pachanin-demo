# Прототип «детски простого» ЛК

`lk-prototype.html` — самодостаточный кликабельный прототип кабинета покупателя
(открывается любым браузером, без сборки). Демонстрирует целевой уровень простоты
из `DESIGN_PLAYBOOK_cabinets_2026-07-16.md`.

## Шесть правил «понятно ребёнку», применённых в прототипе

1. **Одно дело за раз.** Первый экран начинается с приветствия и одной фразы:
   «Сегодня одно важное дело и два обычных». Дело-герой — одна карточка с одной кнопкой.
2. **Язык десятилетнего.** «Банк должен подтвердить, что деньги готовы» вместо
   «резерв ожидает банковского подтверждения». Каждый статус объясняет последствие:
   «Пока банк не ответит, зерно не поедет к вам».
3. **Светофор вместо статусной системы.** Красная точка — нужно ваше действие,
   жёлтая — ждём кого-то другого, зелёная — готово, серая — не срочно.
4. **Деньги — тремя строками.** Свободны / Отложены / На паузе, у каждой строки
   одна причина и одна ссылка «Почему?».
5. **Сделка — как история.** Таймлайн простыми словами («Машину взвесили»,
   «Вы договорились о цене»), сверху самое новое. Профессиональные детали —
   в свёрнутом блоке «Подробности для бухгалтера и юриста».
6. **Действие = обещание.** Экран решения говорит, что произойдёт,
   что не произойдёт («деньги никуда не спишутся») и подтверждает результат
   отдельным экраном.

## Правила «решает человек, а не система» (v2)

7. **Система советует, но не решает.** Экран решения — три варианта с меткой
   «советуем» у рекомендуемого: подтвердить сейчас / напомнить завтра /
   изменить сумму. У каждого варианта написано последствие. Кнопка подтверждения
   меняет текст под выбранный вариант, и «ничего не произойдёт без этой кнопки».
8. **Решение обратимо.** Пока банк не ответил, запрос можно отозвать; напоминание
   можно перенести. Обратимость написана прямо на экране.
9. **Свобода перемещения.** Постоянная навигация слева (Главная · Мои сделки ·
   Деньги · Документы) — человек в любой момент уходит куда хочет, бейдж
   показывает, где ждёт решение. Никакой экран не запирает пользователя.
10. **Переходы предсказуемы.** На глубине больше одного уровня — хлебные крошки
    («Главная / Сделка ДЛ-9106 / Подтверждение денег»), каждая — кликабельна.
    Ссылки «Открыть сделку →» из любого места ведут в одну и ту же карточку.
11. **Грамматика кнопок.** Первичная кнопка = глагол + предмет («Отправить запрос
    банку»), одна на экран, всегда зелёная. Вторичная — контурная («Вернуться
    позже», «Открыть сделку»). Ссылки со стрелкой ведут на другой экран.
    Некликабельные статусы не выглядят кнопками.

Сценарий для проверки: Главная → «Выбрать, что делать» → выбор варианта →
кнопка меняет текст → подтверждение → главная переключается в «Ждём банк»,
бейдж исчезает. Тёмная тема — автоматически от системной настройки.
