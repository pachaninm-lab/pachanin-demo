# Прототип ЛК — рабочий инструмент (v3)

`lk-prototype.html` — самодостаточный кликабельный прототип кабинета покупателя
(открывается любым браузером, без сборки). Целевой уровень: **взрослая рабочая
платформа мирового класса** — плотная и функциональная, но упорядоченная.
Основа — `DESIGN_PLAYBOOK_cabinets_2026-07-16.md`.

## Позиционирование

Не «игрушка с одной кнопкой» и не хаос панелей. Формула:
**плотность данных × строгая иерархия × человеческий язык.**

## Что демонстрирует v3

1. **Профессиональный каркас.** Постоянная боковая навигация с группами
   (Работа / Рынок), строка поиска с горячей клавишей Ctrl K, рабочие разделы:
   Обзор · Сделки · Деньги · Документы (+ заглушки Торги · Контрагенты · Отчёты).
2. **Обзор как рабочий стол, а не витрина.** Ряд KPI (требуют решения / в резерве /
   удержано / активные сделки) → очередь «Требуют вашего решения» со сроками и
   контекстными действиями в строке → лента событий. Никакой кнопки-героя:
   действия распределены по строкам, приоритет задаётся сроком и статусом.
3. **Настоящие таблицы.** Сделки и деньги — плотные data-таблицы: моноширинные
   табличные цифры, выравнивание чисел вправо, колонка «Следующий шаг», статусы
   чипами (решение за вами / ждёт / по графику), фильтры, клик по строке.
   Это «набор строк» с иерархией — а не хаос: у каждой строки есть статус,
   следующий шаг и одно место, куда она ведёт.
4. **Карточка сделки — как в финтех-платформах.** Полоса ключевых фактов,
   вкладки (Хронология / Документы / Деньги / Качество), хронология событий
   с моноширинными таймстампами, справа — панель действий с суммами и
   первичной кнопкой.
5. **Решение остаётся за человеком.** Экран решения: три варианта с меткой
   «Рекомендуем», последствие у каждого, кнопка меняет текст под выбор,
   «действие выполняется только по этой кнопке», решение обратимо.
6. **Деловой язык без канцелярита и без сюсюканья.** «Запросить подтверждение
   резерва», «типичное время ответа — до 2 часов», «спишутся или вернутся строго
   по акту». Никаких «зерно поедет к вам» и никаких «контуров исполнения».
7. **Обратная связь — тостом, не экраном.** Подтверждение действия показывает
   toast и обновляет очередь на месте (строка меняет статус, бейдж уменьшается) —
   как в Stripe/Linear, без прыжка на отдельный экран «Ура, готово».

## Система

- Токены light/dark, плотная типографика 13–14px, KPI 21px;
- суммы и номера — моноширинные с табличными цифрами;
- статусные чипы: act (красный, решение за вами) / wait (жёлтый) / ok (зелёный) / mute;
- кнопки: primary (зелёная, глагол+предмет, одна на контекст) / secondary (контурная) / small (в строках таблиц);
- SVG-иконки в навигации, никаких эмодзи в рабочих экранах;
- хлебные крошки на глубине >1, клик по строке = переход в карточку.

Сценарий проверки: Обзор → «Рассмотреть» в очереди → выбор варианта →
«Запросить подтверждение» → возврат на Обзор: строка очереди сменила статус,
бейдж 2→1, тост подтвердил отправку.
